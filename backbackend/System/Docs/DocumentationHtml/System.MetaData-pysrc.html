  2  Routines for obtaining, generating, and processing meta data about data files,  
  3  file runs, and functions used in the data file generation process.  
  4   
  5  The way that the metadata scheme works is: 
  6   
  7  In the top level System Folder there is a directory ../System/MetaData that  
  8  contains a replica of the file structure outside of the System folder.    
  9  Each "real" path corresponds to a directory in the MetaData directory, 
 10  in which metadata about the path is stored,  
 11  e.g. '../Data/Dan_Data/NPR_Puzzle_Solutions', would correspond to  
 12  '../System/MetaData/Data/Dan_Data/NPR_Puzzle_Solutions'.    
 13  Metadata is similarly attached to functions in python modules referenced via their dot-paths.    
 14           
 15  Given a file path P, the associated path for the metadata is given by the  
 16  function metadatapath(P).  Given a python function dot path, the associated  
 17  path P for the metadata is given by the function opmetadatapath(P).    
 18   
 19  Inside a given path's metadata directory is a variety of associated metadata files.  
 20  There are three main kinds of associated things: 
 21          -- data generated during the runtime that produced/was generated by the path.  
 22          -- metadata attached by the human directly to the file.   
 23          -- data used and generated by the system Graphical Browser.   
 24          e.g. Linklists local to the path, and graphviz-generated graphs of these LinkLists.
