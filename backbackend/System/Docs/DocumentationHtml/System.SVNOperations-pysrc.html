  1  ''' 
  2  Routines for running the "flat" SVN archive that allows storage of  
  3  files w/o .svn directories.  
  4   
  5  The basic idea is that:  files that the user wants to put in SVN are copied,  
  6  with a naming convention, to a single large flat SVNArchive directory.    
  7  Then, files in this directory are actually the ones put under SVN control.     
  8   
  9  This module provides convenience functions for managing this process,  
 10  including CheckIn(), Update(), and Status().  
 11   
 12  Each of these functions has 3 basic steps: 
 13   
 14  1) before doing anything, the archive is "Equalized", meaning that --  
 15  at least for the directories the user has specified in a configuration file --  
 16  the achive is made to contain the exact same contents as the user's real directories.      
 17  The user's file system is always treated with priority -- whatever the contents there,  
 18  that is what the archive is made to reflect.  
 19   
 20  2) then the SVN action occurs, be it Check In or Update, or Status.  
 21   
 22  3) the the users' real file system is Equalized with the archive, to get any new  
 23  or modified files or delete those that have been removed in the SVN repository.  
 24   
 25  To use these routines the environment variable 'SVNArchivePath' should be set 
 26  -- this reflects the location of the SVNArchive.   
 27   
 28  Each of them takes an optional SVNPath argument, which can be used  
 29  in place of the users' specified confiuration file.    
 30  '''
